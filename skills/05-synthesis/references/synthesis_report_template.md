# 第 XX 轮综合分析报告

> 本报告不是变更日志，而是对关键问题的**深度分析文档**。
> 写的时候问自己：读完这轮论文后，我对这个领域的理解到底**深化**了什么？
> 不是"我读了什么"，而是"我现在知道了什么、为什么知道、这对课题意味着什么"。

---

## 1. 本轮调研背景

**本轮想回答的核心问题**（来自 round_goal.md）：

用 1-2 段话讲清楚：
- 为什么本轮选择这些论文/方向？（在 long_plan 中的位置）
- 本轮调研在整个调研路线中的角色是什么？（确认 gap / 建立基线 / 探索方向 / 补充细节）
- 精读了哪些论文（简要列表）以及每篇为什么入选

然后自评：本轮是否回答了想回答的问题？如果部分回答/没回答，为什么？

---

## 2. 核心发现与分析

**这是报告最重要的部分。不允许写成要点罗列。** 必须是成段的分析叙述。

对每条重要发现，按以下结构展开，每条 200-500 字的分析：

**发现 X：[一句话核心发现]**

- **证据**：来自哪（几）篇论文的什么实验数据/结论/方法
- **分析**：这个发现意味着什么——它改变了之前对领域的什么理解？它确认或推翻了什么假设？它揭示了什么新问题？
- **对课题的影响**：这个发现如何影响本课题的方向判断？（确认了 gap 存在 / 缩小了创新空间 / 提供了技术线索 / 引出了新风险）

如果多条发现之间有逻辑关联（如多个工作从不同角度指向同一结论），必须点明交叉印证关系，而不是孤立地罗列。

---

## 3. 对现有工作的重新评估

不是"新增了哪些工作"的列表，而是**对本轮精读后整个领域理解的变化**的分析。

- 哪些工作现在看来比之前评估的更重要？为什么？
- 哪些工作的定位发生了变化（从"竞争对手"变为"基线参考"、"从"可能相关"升级为"核心参考"）？
- 本轮精读后，对领域的方法格局有什么新认识？（流派的此消彼长、新范式的出现、某条路线的天花板被确认）
- 最近邻工作矩阵的更新（如有变化）

---

## 4. 对 gap 和不足的分析

对每个重要 gap 进行**逐项深度分析**，不是一句话带过。每条 gap 按以下结构：

**Gap X：[具体描述]**

- **本轮证据更新**：本轮论文如何支撑或质疑这个 gap？
- **证据强度变化**：（强→强 / 弱→中 / 从假设升级为确认 / 从确定降级为存疑）
- **具体化程度**：这个 gap 现在能多具体地描述？（场景 + 约束 + 后果 + 为什么现有方法解决不了）
- **对课题的影响**：这个 gap 是否适合作为申请书的核心切入点？为什么适合/不适合？

同时分析：
- 哪些之前认为存在的 gap 被本轮证据削弱或排除了？
- 有没有发现新的、之前没意识到的 gap？

---

## 5. 对可能突破方向的分析

不是"新增/删除"的清单操作，而是**对每个候选方向的深度评估**：

**方向 X：[技术思路]**

- **本轮启发来源**：哪些论文的具体方法/结论启发了这个方向？
- **可行性更新**：本轮证据是否增强了/削弱了这个方向的可行性？
- **差异化分析**：这个方向与已有工作的差异是否足够清晰、足够有说服力？
- **验证可行性**：能否在项目周期内验证这个方向的主张？
- **风险更新**：新发现了什么风险或不确定性？

## 6. current_view.md 变更说明

本节是 current_view.md 的变更索引——帮助后续读者快速定位变化。**每条变更必须标注具体影响了 current_view.md 的哪一节**。

格式：

```
[current_view §二.X] 新增方向"XXX"——来源：论文A + 论文B 的交叉印证
[current_view §三.X] Gap X 证据强度从"中"升级为"强"——来源：论文C 的 XXX 实验
[current_view §四.X] 新增候选方向——来源：论文D 的方法启发 + 论文E 的局限性反推
[current_view §五] 课题创新性置信度从"中"升级为"高"——依据：X篇最近邻工作均未覆盖共享约束
```

如果本轮推翻了 current_view 中的旧判断，单独标注：

```
[推翻] current_view §X.Y 中原判断"XXX"——推翻依据：论文F 的 XXX 数据表明原假设不成立
```

## 7. 跳过与优先级建议

对 long_plan 中尚未执行的任务给出基于本轮发现的优先级调整建议。每条建议必须给出**本轮证据依据**：

- **建议跳过**：[任务 ID] — 理由：本轮论文 X/Y/Z 已充分回答该任务要验证的问题；继续查该方向边际收益低
- **建议降级**：[任务 ID] — 理由：本轮发现该方向对核心论证主线的影响小于预期
- **建议新增**：[方向] — 理由：本轮发现了新的 gap/方向，需要专门调研

## 8. 下一步建议

**是否建议进入 helm**：是 / 否 / 条件性

给出明确判断依据（引用本轮的具体发现），不是泛泛的"还需要继续调研"：

- 如果建议进入 helm：核心 gap 确认到了什么程度？最近邻覆盖是否充分？继续调研的边际收益是否递减？还有哪些细节可以在 helm 阶段按需追溯？
- 如果建议继续调研：还缺少哪个关键问题的答案？为什么这个问题的答案对 helm 是必需的（而非 nice-to-have）？下一轮最优先的方向和论文。
- 如果条件性进入：什么条件满足后可以进入？这个条件能否在 1 轮内解决？
